Animated Gameboy in CSS
This is the technical blog post. Click here to see the actual demo.
   

Being able to draw with CSS is knowing what tools and techniques you have at your disposal to leverage and abuse.

Of course, once you load yourself with a wide enough CSS arsenal, you enable the ability to implement certain features in a myriad of ways. Therefore for this project, I made it a priority to use as few <div> elements as possible without sacrificing the desired effect.

   Below are five advanced techniques that I've found useful for doodling in CSS.
1. Make use of letters and fonts for shapes and objects
   Fonts provide a range of characters and that we can leverage to create interesting shapes and edges. font-size, text-shadow, and letter-spacing are amongst several animatable properties that can style letters. Together, they can be used to create intricate patterns not possible (or extremely difficult) with regular CSS techniques. The legs in the processor chip are actually just a string of "l"'s that have been rotated!

2. Animating text with a linear-gradient (webkit only, with fallback)
   Currently, only webkit browsers have CSS support for creating text with a linear gradient fill. In short, it involves setting a linear-gradient on your element background, while applying -webkit-background-clip: text to mask away the non text background, and finally using -webkit-text-fill-color: transparent to let the gradient become the font-color.

   Since webkit properties only work on selected browsers, the above technique will cause non webkit vendors to degrade and leave behind text on a undesireable linear gradient background.

   However, thanks to this clever trick, you can degrade gracefully by using -webkit-linear-gradient to force a gradient background for webkit browsers only, then set color as a cross browser fallback. From here, you can animate the gradient by using background-position to achieve the desired text animation.

5. Identify repeatable shapes and simplify
   Recognize when symmetrical or complex objects can be broken down into repeatable, smaller components. Style and animate the simpler element, then duplicate, translate, and rotate through CSS transform's to recreate the full complex original.

   Keep in mind that all CSS transform's use transform-origin as the reference point for applying the animation. By default, this (x, y, z) value is set to 50% 50% 0, which is the midpoint of the element in the current z-index plane.

   The default origin is ideal in most cases when your point of symmetry is dead center of the element (i.e when you're trying to mirror a block of text, or when you're trying to animate a spinner). But if you're constructing an object by fitting together rotated components, make sure that the origin is set to the rotational point of symmetry.

   Drawing with CSS is a very tedious task of eyeballing positions and tweaking element properties, in order to achieve that pixel perfect curve. There's no shortcuts or automated tools (yet) to help generate CSS for intricate drawings loaded with animations.

   ...But not that there ever needs to be, because from a Time Spent vs Outcome perspective, it is definitely unfavorable compared to traditional (and digital) pen and paper methods. Hours of work spent tweaking and tweening CSS can easily be replicated in swift minutes using photoshop or other similar means.

   That being said, learning and drawing with CSS allows you the opportunity to dive deep and explore the vast specification that defines everything we see on the web. It gives you a chance to familiarize yourself with current established properties, while suprising you with bleeding edge features that you had no idea existed. This knowledge is invaluable for achieving desirable UX and meeting design expectations in a quick and performant manner, whether you're building a production scale web app or tinkering a personal web tool.

   All in all, do what you love, and never stop learning.
