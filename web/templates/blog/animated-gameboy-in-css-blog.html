5. Rotational symmetry with transform-origin
All CSS transform's use transform-origin as the reference point for applying the transformation. By default, this (x, y, z) value is set to 50% 50% 0, which is the midpoint of the element in the current z-index plane. This is ideal in most cases when your point of symmetry is dead center of the element (i.e when you're trying to mirror a block of text, or when you're trying to animate a spinner).

However, when you're aiming to create rotational symmetry with asymmetric elements, not setting the proper transform-origin will induce a ripple effect and cause your animations to run askew, since everything is animating off the element instead of the desired rotational center.
